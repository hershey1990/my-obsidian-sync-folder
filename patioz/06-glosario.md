# 📖 Glosario de Patioz — Lenguaje Ubicuo

## 📌 Dominio de Real Estate

| Término | Definición |
|---|---|
| **Propiedad / Inmueble** | Unidad inmobiliaria individual (casa, departamento, terreno, local comercial) |
| **Cliente** | Persona o entidad que usa la plataforma para buscar o gestionar propiedades |
| **Dueño / Propietario** | Titular legal del inmueble |
| **Inquilino / Arrendatario** | Persona que alquila la propiedad |
| **Contrato** | Acuerdo legal entre propietario e inquilino (arrendamiento, compra-venta) |
| **Comisión** | Porcentaje o monto fijo que cobra la plataforma por cada transacción |
| **Zona / Barrio** | Área geográfica dentro de una ciudad |
| **Polígono** | Delimitación geográfica (GeoJSON) de una zona en el mapa |
| **Publicación / Listing** | Anuncio de una propiedad disponible en la plataforma |
| **Visita / Tour** | Cita programada para mostrar una propiedad a un cliente potencial |
| **Solicitud** | Petición de un cliente para contactar al dueño o agendar una visita |

## ⚙️ Dominio Técnico

| Término | Definición |
|---|---|
| **BFF** | Backend for Frontend — capa de orquestación entre el frontend y los microservicios |
| **Auth API** | Servicio de autorización basado en roles (RBAC) sobre NestJS + Supabase |
| **imgproxy-api** | Microservicio de subida de archivos a S3 con procesamiento de imágenes |
| **MapUI** | Frontend principal con mapa interactivo (Next.js + Leaflet) |
| **QStash** | Cola de mensajería asíncrona (Upstash) para orquestación de tareas |
| **localStash** | Simulador local de QStash para desarrollo |
| **GeoJSON** | Formato estándar para datos geográficos (polígonos, puntos) |
| **Clean Architecture** | Principio de diseño con separación estricta entre dominio, aplicación e infraestructura |
| **RBAC** | Role-Based Access Control — control de acceso basado en roles |
| **MOC** | Map of Content — nota índice que agrupa y organiza un tema |

> *Actualizar este glosario cada vez que surja un término nuevo del dominio o del sistema.*
