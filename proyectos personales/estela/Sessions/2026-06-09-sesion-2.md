# Sesión 2 — 2026-06-09

## Qué hicimos
1. Diseño completo de arquitectura frontend para el Marketplace de Autos
2. Evaluación de tradeoffs: app única vs apps separadas
3. Decisión: monorepo con Turborepo, 2 apps Next.js (web + admin)
4. ADR-003 creado con la arquitectura detallada

## Decisiones clave

| Decisión | Valor |
|----------|-------|
| Estructura repo | Monorepo (Turborepo) |
| Apps | `apps/web` (estela.com) + `apps/admin` (admin.estela.com) |
| Framework | Next.js 14+ App Router (ambos) |
| UI | Tailwind + shadcn/ui via `@estela/ui` |
| Data fetching | RSC (público) + TanStack Query (dashboards) |
| Estado cliente | Zustand (mínimo: auth, search, compare, UI) |
| Auth | BFF proxy, httpOnly cookies, middleware por rol |
| Dealer dashboard | Integrado en `apps/web` (`/dealer/*`) |
| Publish flow | Multi-step wizard |
| Upload fotos | Via Laravel proxy |
| Packages compartidos | `@estela/ui`, `@estela/types`, `@estela/utils`, `@estela/config` |
